## 🏃 주제 C. 스포츠/마라톤 이벤트 관리 플랫폼 (Run Together)

- **특징:** 지도 API 기반 코스 시각화 중심의 러닝·마라톤 이벤트 관리 및 참가자 기록 처리
- **역할 분담:**
  - **팀원 1:** 회원 인증(JWT), 마이페이지(참가 이력/기록 모음), 러너 프로필 관리
  - **팀원 2:** 이벤트 CRUD, 거리별/지역별/날짜별 필터링, 참가 신청/취소 로직
  - **팀원 3:** 카카오맵 API 연동, 코스 경로 시각화(폴리라인), 코스 좌표 관리 어댑터
  - **팀원 4:** 기록 입력/관리(CSV 일괄 업로드), 랭킹 보드, 완주 인증서 PDF 생성(jsPDF)
  - **팀원 5:** 참가/기록 통계 대시보드(Chart.js), 관리자 페이지(이벤트/회원/기록 관리)

### 1. 마라톤·러닝 이벤트의 세부 의미

- **풀 마라톤(Full Marathon):** 42.195km를 달리는 공식 거리. 국제육상경기연맹(IAAF)이 정한 표준 거리. 엘리트 선수 기준 약 2시간, 일반인 기준 4~6시간 소요.
- **하프 마라톤(Half Marathon):** 21.0975km. 풀 마라톤의 절반 거리로, 초보 러너가 도전하기 적합한 중간 단계.
- **10K / 5K:** 각각 10km, 5km 코스. 러닝 입문자부터 숙련자까지 부담 없이 참여 가능한 대중적 거리.
- **페이스(Pace):** 1km를 달리는 데 걸리는 시간(분:초). 예: 5'30" 페이스 = 1km당 5분 30초. 러너의 실력을 나타내는 핵심 지표.
- **BIB 번호(배번호):** 참가자에게 부여되는 고유 번호. 기록 측정, 사진 검색, 본인 확인에 사용.
- **에이드 스테이션(Aid Station):** 코스 중간에 설치된 급수·급식 지점. 코스 설계 시 반드시 포함.
- **컷오프 타임(Cut-off Time):** 제한 시간. 해당 시간 내에 완주하지 못하면 기록 미인정.

---

### 2. 플랫폼 핵심 기능 및 목적

- **배경:** 국내 러닝 인구 1,000만 명 시대. 나이키 런 클럽, NBRC 등 러닝크루 문화가 확산되면서 소규모 러닝 이벤트부터 대규모 마라톤 대회까지 개최 수요가 급증. 기존 마라톤 접수 사이트(마라톤온라인, 런닝포탈)는 UX가 구식이고 코스 시각화·기록 관리 기능이 부실.
- **목적:** 러닝·마라톤 이벤트를 개설하고, 코스를 지도 위에 시각화하며, 참가 접수·기록 관리·랭킹·완주 인증서를 통합 제공하여, 주최자와 러너 모두를 위한 원스톱 플랫폼 구축.
- **Cloud Native 연결:** 대회 접수 기간에 트래픽이 몰리므로, Docker 컨테이너 기반 배포로 탄력적 대응. 지도 타일 캐싱은 Redis로 처리하여 성능 최적화.

### 3. 핵심 기술 포인트

- **지도 API 연동:**
  - 카카오맵 JavaScript API + react-kakao-maps-sdk 활용
  - 폴리라인(Polyline)으로 코스 경로를 지도 위에 시각화
  - 마커(Marker)로 출발점, 반환점, 에이드 스테이션, 도착점 표시
  - 관리자가 지도 위 클릭으로 경로점 추가 → 자동 거리 계산
- **기록 관리:**
  - 운영자가 참가자별 완주 시간 입력 (수동 + CSV 일괄 업로드)
  - 자동 페이스 계산 (완주 시간 ÷ 거리)
  - 대회별/전체/거리별 랭킹 자동 생성
- **완주 인증서 PDF:**
  - jsPDF 라이브러리로 클라이언트 사이드 PDF 생성
  - 대회명, 참가자 이름, BIB 번호, 완주 기록, 순위 포함
