## 1. 🎯 프로젝트 개요

### 1-1. 프로젝트명
> **Screening Room** — 온라인 영화제·공모전 관리 플랫폼

### 1-2. 프로젝트 배경 및 동기

- 유튜브/틱톡 등 1인 크리에이터 시대, 단편영화·영상 공모전이 급증하고 있지만 통합 관리 플랫폼은 부족
- 부산국제영화제(BIFF) 같은 대형 영화제도 온라인 상영·투표를 병행하는 추세
- "이벤트(영화제/공모전) 관리 플랫폼"으로 IBM 주제에 부합하면서 독창적인 포트폴리오 구현 가능

### 1-3. 프로젝트 목표

- **핵심 목표**: 영화제/공모전을 개설하고, 작품 출품·온라인 상영·관객 투표·심사를 관리하는 Cloud Native 플랫폼 구축
- **세부 목표 1**: 영화제 개설 및 작품 출품/관리 기능 구현
- **세부 목표 2**: 관객 실시간 투표 및 심사 시스템 구현
- **세부 목표 3**: 통계 대시보드 및 수상 결과 발표 기능 구현

### 1-4. 타겟 사용자

| 사용자 유형 | 설명 | 주요 니즈 |
|------------|------|----------|
| 관객(일반 사용자) | 영화제에서 작품을 감상하고 투표하는 사용자 | 작품 탐색, 감상, 투표, 리뷰 작성 |
| 출품자(크리에이터) | 작품을 출품하는 영상 제작자 | 작품 등록, 출품 현황 확인, 심사 결과 확인 |
| 심사위원 | 작품을 심사하는 전문가 | 작품 열람, 심사 기준별 점수 입력 |
| 운영자 | 영화제를 기획·운영하는 주최측 | 영화제 개설, 작품/심사/일정 관리, 수상작 발표 |
| 시스템 관리자 | 플랫폼 전체 관리 | 회원/이벤트 관리 |

## 3. 🔍 프로젝트 방향성

### 3-1. 핵심 컨셉 (한 줄 요약)
> **"크리에이터가 출품하고, 관객이 투표하며, 심사위원이 평가하는 온라인 영화제 플랫폼"**

### 3-2. 차별화 포인트

1. **관객 실시간 투표** — WebSocket 기반 실시간 관객 인기투표, 득표 현황 차트 라이브 업데이트
2. **이중 심사 체계** — 심사위원 전문 평가 + 관객 투표를 가중 합산하여 수상작 결정
3. **출품 → 심사 → 수상의 완전한 워크플로우** — 영화제 라이프사이클 전체를 관리

### 3-3. 참고 서비스 / 벤치마킹

| 서비스명 | URL | 참고할 점 | 개선할 점 |
|---------|-----|----------|----------|
| Vimeo (영화제 채널) | https://vimeo.com | 영상 재생 UX, 크리에이터 프로필 | 영화제 관리 기능 없음 |
| FilmFreeway | https://filmfreeway.com | 영화제 출품 프로세스, 수상 관리 | 영어 전용, 국내 UX 부족 |
| BIFF (부산영화제) | https://biff.kr | 일정 관리, 프로그램 구조 | 관객 투표 기능 제한적 |

### 6-2. 화면 흐름도 (User Flow)

```
[관객 Flow]
메인 페이지 → 회원가입/로그인 → 영화제 목록 → 영화제 상세 → 작품 목록
                                                          → 작품 상세 → 영상 감상
                                                                     → 투표
                                                                     → 리뷰 작성
                              → 투표 현황 (실시간 차트)
                              → 수상 결과 확인

[출품자 Flow]
출품자 로그인 → 작품 출품 (YouTube URL + 정보 입력) → 출품 현황 확인
                                                  → 심사 결과 확인

[심사위원 Flow]
심사위원 로그인 → 배정된 작품 목록 → 영상 감상 → 기준별 점수 입력

[운영자 Flow]
운영자 로그인 → 영화제 개설 → 심사 기준 설정 → 심사위원 배정
                                            → 출품작 관리 (승인/반려)
                                            → 수상작 확정 및 발표
```

## 7. 📊 데이터 설계 (ERD 초안)

### 7-1. 주요 엔티티(테이블) 목록

| # | 엔티티명 | 설명 | 주요 속성 |
|---|---------|------|----------|
| 1 | User | 사용자 정보 | id, email, password, name, role(AUDIENCE/CREATOR/JUDGE/ORGANIZER/ADMIN), profile_image, created_at |
| 2 | Festival | 영화제 정보 | id, title, description, theme, submission_start, submission_end, voting_start, voting_end, status(SUBMISSION/SCREENING/VOTING/COMPLETED), created_by(FK) |
| 3 | Film | 출품작 | id, festival_id(FK), creator_id(FK), title, description, genre, youtube_url, poster_image, duration, status(PENDING/APPROVED/REJECTED), submitted_at |
| 4 | AudienceVote | 관객 투표 | id, film_id(FK), user_id(FK), voted_at |
| 5 | JudgeCriteria | 심사 기준 | id, festival_id(FK), criteria_name, weight(가중치), max_score |
| 6 | JudgeScore | 심사 점수 | id, film_id(FK), judge_id(FK), criteria_id(FK), score, comment |
| 7 | Award | 수상 내역 | id, festival_id(FK), film_id(FK), award_type(GRAND/AUDIENCE/SPECIAL), announced_at |
| 8 | Review | 관객 리뷰 | id, film_id(FK), user_id(FK), content, rating, created_at |

### 7-2. 엔티티 관계

- User 1:N Film — 한 크리에이터가 여러 작품 출품
- Festival 1:N Film — 한 영화제에 여러 출품작
- Festival 1:N JudgeCriteria — 한 영화제에 여러 심사 기준
- Film 1:N AudienceVote — 한 작품에 여러 관객 투표
- Film 1:N JudgeScore — 한 작품에 여러 심사위원 점수
- Film 1:N Review — 한 작품에 여러 리뷰
- Festival 1:N Award — 한 영화제에 여러 수상 내역
