============================================================
  IBM 박성식 특강 요약 - 프로젝트 안내
============================================================

■ 특강 개요
──────────────────────────────────────
- 강사: IBM 박성식
- 대상: 하이비디오 기본 교육 수강생
- 목적: 수행해야 할 2개의 프로젝트 사례 소개 및 프로젝트 진행 절차 안내
- 프로젝트 구성:
  · 첫 번째 프로젝트: 2개의 사례 중 1개를 선택하여 진행
  · 두 번째 프로젝트: 자유 주제로 진행


============================================================
  프로젝트 1: 클라우드 네이티브 통합 이벤트(포럼) 플랫폼 관리
============================================================

■ 프로젝트 소개
──────────────────────────────────────
- IBM이 실제 수행했던 프로젝트
- 국내 대기업에서 '세계경제포럼(다보스포럼, WEF)' 을 참고하여
  국내에서도 유사한 포럼 시스템을 만들고자 진행한 프로젝트
- 유명인사·강사를 초빙하여 세미나(포럼)를 진행하고,
  포럼 정보를 보여주며 참석·관리하는 시스템

■ 주요 기능
──────────────────────────────────────

[포럼 전 기능]
- 사용자 등록 / 로그인
- 사전 등록 및 포럼 정보 확인
- 앞으로 진행될 프로그램·일정표 확인
- 연사 및 아젠다 소개
- 지난 포럼 다시보기(과거 포럼 확인)

[포럼 중 기능]
- 현재 진행 중인 포럼 실시간 확인 및 참여
- 온라인 스트리밍을 통한 참석 (Zoom 연동)
- 실시간 투표, 댓글, Q&A
- 양방향 토론 기능
- 실시간 성과 분석
- 라이브 블로그, 키 모먼트 표시

[포럼 후 기능]
- 설문조사
- 사후 성과 분석
- 지난 포럼 다시보기 및 하이라이트 세션 모아보기

[공통 기능]
- 참여 현황 그래프 시각화
- 일자별·주제별 소개 페이지 및 필터링
- 카드 형태 UI
- 알림·공지 기능
- 마이페이지, 세션 추천, 통합검색

[관리자 기능]
- 회원 관리
- 세션 등록 및 공지
- 사회자 지정
- 시스템·사이트 관리
- 통계 및 결과 조회

■ 기술 아키텍처
──────────────────────────────────────
- 마이크로서비스 아키텍처(MSA) 기반
- CI/CD 파이프라인 구성
- 클라우드 환경 + Docker 컨테이너
- DB: MariaDB (마스터-슬레이브 이중화 구성)
- 캐시/세션: Redis (클러스터 구성)
- 서버: JBoss + Spring Boot 기반
- JDK 사용
- 클라이언트(화면): jQuery 기반 (현재는 React 등 추천)
- 사용자용 서버와 관리자(EA)용 서버 분리 구성
- 포트포워딩, 온라인 회의 연동 (Zoom)

■ 참고 사이트
──────────────────────────────────────
- 세계경제포럼(WEF): www.weforum.org
- SK 이천 포럼 사이트 (접속에는 계정/권한 필요)


============================================================
  프로젝트 2: 코리빙(Co-Living) 플랫폼
============================================================

■ 프로젝트 소개
──────────────────────────────────────
- 공동 거주 환경에서 IoT 기기를 연동하는 플랫폼
- IoT 기기 제공 업체와 공동 거주 서비스 업체 사이의 미들웨어(중간 플랫폼) 역할
- 공동 거주 업체가 IoT 기기(청소기, 세탁기, 공용 현관 등)와
  쉽게 연결할 수 있도록 플랫폼을 제공

■ 주요 기능
──────────────────────────────────────

[게이트웨이 서비스 기능]
- API Gateway 역할: IoT 서비스 API를 중계
- 요청 라우팅, API 관리, 보안(인증/권한), 모니터링
- 다양한 IoT 기기 API를 하나의 통합 API로 제공
- 노코드/로우코드 기반으로 연기 확장 가능

[어플리케이션 서비스 기능]
- 거주자용 어플리케이션 서비스 제공
- IoT 기기 확인·제어 기능

[어드민(관리자) 기능]
- 고객사 관리
- 서비스 관리
- 공유 면적 관리
- 기기 등록·관리
- 모니터링 관리
- 시스템 권한 관리

■ 기술 아키텍처
──────────────────────────────────────
- 컨테이너 기반 (Amazon EKS)
- API Gateway 패턴 적용
- VPC 네트워크를 통한 IoT 업체 연결
- 인터넷 로드밸런서 → EKS 서비스

■ 참고 사이트
──────────────────────────────────────
- 삼성전자 SmartThings Developer 사이트
  (IoT API 참고용, 실제 디바이스 없이도 목업 서비스로 대체 가능)


============================================================
  API Gateway 개념 설명
============================================================
- 뒷단의 복잡한 여러 시스템/서비스 앞에 하나의 API Gateway를 두어
  사용자(컨슈머)는 하나의 엔드포인트만으로 다양한 서비스 이용 가능
- 주요 기능: 요청 라우팅, API 관리, 보안(인증/권한),
  모니터링, 로깅, 과금 관리 등
- MSA 구성 시 필수적으로 사용됨
- 대외 서비스용 API 제공 시에도 반드시 필요


============================================================
  프로젝트 진행 시 주의사항 (박성식 강조)
============================================================

1. 범위 축소 필수
   - 두 프로젝트 모두 실제 수행 시 적지 않은 인력과 긴 기간이 소요된 프로젝트
   - 교육 후 프로젝트 팀 인원과 기간에 비해 규모가 매우 크므로,
     전체 범위를 수행하는 것은 현실적으로 어려움
   - 조별로 잘 검토하여 적정한 범위를 선정해야 함

2. 구현 제외 추천 영역
   - 포럼 프로젝트: Zoom 연동(실시간 스트리밍) 구현은 어려우므로 제외 추천
   - 코리빙 프로젝트: 실제 IoT 디바이스가 없으므로
     목업(mock) 서비스나 대체 서비스를 사용하여 구현할 것을 추천

3. 프로젝트 진행 단계
   - 프로젝트 기획 및 설계
   - 데이터 관리 및 전처리
   - 백엔드 개발
   - 프론트엔드 개발
   - CI/CD 배포
   - 테스트 및 최적화
   - 프로젝트 마무리 및 분석

4. 멘토링 활용
   - 멘토링을 통해 실제 구현 범위를 결정하는 것을 권장

5. 프레임워크 선택
   - 화면(프론트엔드)은 교육에서 배운 프레임워크를 선택하여 개발
   - 요즘은 React 등을 많이 사용
