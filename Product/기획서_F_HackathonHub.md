## 1. 🎯 프로젝트 개요

### 1-1. 프로젝트명
> **Hackathon Hub** — 해커톤 관리 플랫폼

### 1-2. 프로젝트 배경 및 동기

- 해커톤(Hack + Marathon)은 제한된 시간 내에 팀을 이루어 프로토타입을 만들어 발표하는 대회형 이벤트
- 삼성, 카카오, 네이버, NASA 등 국내외 기업·기관에서 해커톤을 활발히 개최하는 추세
- IBM 원본 프로젝트(포럼 플랫폼)와 구조가 유사 — "세션→세미나→참가→피드백"을 "해커톤→팀빌딩→개발→심사"로 치환
- 개발자 대상 이벤트이므로 포트폴리오에서 면접관에게 즉각적으로 어필 가능

### 1-3. 프로젝트 목표

- **핵심 목표**: 해커톤을 개설하고, 팀 빌딩·프로젝트 제출·멘토링·심사를 관리하는 Cloud Native 플랫폼 구축
- **세부 목표 1**: 해커톤 개설 및 참가자 팀 빌딩 기능 구현
- **세부 목표 2**: 프로젝트 제출 및 심사 시스템 구현
- **세부 목표 3**: 실시간 공지/타이머 + 통계 대시보드 구현

### 1-4. 타겟 사용자

| 사용자 유형 | 설명 | 주요 니즈 |
|------------|------|----------|
| 참가자 | 해커톤에 참가하는 개발자/디자이너/기획자 | 해커톤 탐색, 참가 신청, 팀 구성, 프로젝트 제출, 결과 확인 |
| 멘토 | 참가 팀에 기술적 조언을 제공하는 전문가 | 배정된 팀 확인, 멘토링 요청 응답 |
| 심사위원 | 제출된 프로젝트를 평가하는 전문가 | 프로젝트 열람, 심사 기준별 점수 입력 |
| 운영자 | 해커톤을 기획·운영하는 주최측 | 해커톤 개설, 참가자/팀/멘토/심사위원 관리, 타이머, 통계 |
| 시스템 관리자 | 플랫폼 전체 관리 | 회원/이벤트 관리 |

## 3. 🔍 프로젝트 방향성

### 3-1. 핵심 컨셉 (한 줄 요약)
> **"해커톤의 모든 것을 한 곳에서 — 팀 빌딩부터 심사까지 관리하는 Cloud Native 플랫폼"**

### 3-2. 차별화 포인트

1. **팀 빌딩 시스템** — 참가자가 역할/기술 스택 기반으로 팀원을 모집하거나, 운영자가 랜덤 매칭
2. **프로젝트 제출 + 심사 워크플로우** — 제출 → 심사 → 결과 발표의 완전한 라이프사이클
3. **실시간 카운트다운 & 공지** — WebSocket으로 해커톤 남은 시간 + 운영 공지를 실시간 전달

### 3-3. 참고 서비스 / 벤치마킹

| 서비스명 | URL | 참고할 점 | 개선할 점 |
|---------|-----|----------|----------|
| Devpost | https://devpost.com | 프로젝트 제출 UX 우수, 해커톤 목록 | 영어 전용, 팀 빌딩 기능 약함 |
| MLH (Major League Hacking) | https://mlh.io | 학생 해커톤 허브, 일정 관리 | 해커톤 개설 기능 없음 |
| Gather Town | https://gather.town | 가상 공간 소통 | 해커톤 관리 기능 없음 |

### 6-2. 화면 흐름도 (User Flow)

```
[참가자 Flow]
메인 페이지 → 회원가입/로그인 → 해커톤 목록 → 해커톤 상세 → 참가 신청
                                                        → 팀 빌딩 → 팀 생성 or 팀 합류
                                                        → 해커톤 진행 중: 타이머/공지 확인
                                                        → 프로젝트 제출
                              → 프로젝트 갤러리 → 프로젝트 상세
                              → 결과 발표 확인

[심사위원 Flow]
심사위원 로그인 → 배정된 해커톤 → 제출 프로젝트 목록 → 프로젝트 열람 → 점수 입력

[운영자 Flow]
운영자 로그인 → 해커톤 개설 → 심사 기준 설정 → 심사위원 배정
                                            → 참가자/팀 관리
                                            → 해커톤 시작 (타이머 시작)
                                            → 공지 전송
                                            → 제출 마감
                                            → 심사 결과 확정 → 수상팀 발표
```

## 7. 📊 데이터 설계 (ERD 초안)

### 7-1. 주요 엔티티(테이블) 목록

| # | 엔티티명 | 설명 | 주요 속성 |
|---|---------|------|----------|
| 1 | User | 사용자 정보 | id, email, password, name, role(PARTICIPANT/MENTOR/JUDGE/ORGANIZER/ADMIN), tech_stack(JSON), bio, profile_image, created_at |
| 2 | Hackathon | 해커톤 정보 | id, title, description, theme, rules, max_team_size, min_team_size, start_time, end_time, submission_deadline, status(UPCOMING/TEAM_BUILDING/IN_PROGRESS/JUDGING/COMPLETED), created_by(FK) |
| 3 | Team | 팀 정보 | id, hackathon_id(FK), team_name, description, looking_for(JSON: 모집 역할), is_open, created_by(FK) |
| 4 | TeamMember | 팀원 | id, team_id(FK), user_id(FK), role_in_team(LEADER/MEMBER), joined_at |
| 5 | Recruitment | 팀원 모집글 | id, team_id(FK), wanted_role, wanted_skills(JSON), description, is_active |
| 6 | Project | 제출 프로젝트 | id, hackathon_id(FK), team_id(FK), title, description, github_url, demo_url, screenshots(JSON), tech_used(JSON), submitted_at |
| 7 | JudgeCriteria | 심사 기준 | id, hackathon_id(FK), criteria_name, weight, max_score |
| 8 | JudgeScore | 심사 점수 | id, project_id(FK), judge_id(FK), criteria_id(FK), score, comment |
| 9 | Announcement | 운영 공지 | id, hackathon_id(FK), title, content, created_at, created_by(FK) |
| 10 | Award | 수상 내역 | id, hackathon_id(FK), project_id(FK), team_id(FK), award_type(GRAND/2ND/3RD/SPECIAL), announced_at |

### 7-2. 엔티티 관계

- User 1:N TeamMember — 한 사용자가 여러 해커톤에서 팀으로 활동
- Hackathon 1:N Team — 한 해커톤에 여러 팀
- Team 1:N TeamMember — 한 팀에 여러 팀원
- Team 1:1 Project — 한 팀이 하나의 프로젝트 제출
- Team 1:N Recruitment — 한 팀이 여러 모집글
- Hackathon 1:N JudgeCriteria — 한 해커톤에 여러 심사 기준
- Project 1:N JudgeScore — 한 프로젝트에 여러 심사 점수
- Hackathon 1:N Announcement — 한 해커톤에 여러 공지
- Hackathon 1:N Award — 한 해커톤에 여러 수상
