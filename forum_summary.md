## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | Cloud Native 기반 통합 이벤트 관리 플랫폼 |
| **배경** | IBM이 실제 수행한 프로젝트. 국내 대기업(B사)이 **세계경제포럼(다보스포럼, WEF)** 을 참고하여 온·오프라인 세미나 및 포럼 관리 시스템을 개발 |
| **핵심 컨셉** | 고객사가 요청한 행사 안내, 세션 등록, 온라인 웨비나·세미나 중 투표 및 토론 기능 제공. 통계·인포그래픽으로 행사 데이터 시각화. SNS 연동 알림, 세미나 등록, API 기반 화상 솔루션 연동, 사회자 지정 등의 관리자 기능 포함 |
| **수행 범위** | PC/Mobile 기반의 UI 화면 설계, 디자인 및 참여 단계별 기능 개발 |
| **참고 사이트** | [세계경제포럼(WEF)](https://www.weforum.org), SK 이천 포럼 사이트 (계정/권한 필요) |

## 2. 주요 기능 (포럼 전/중/후)

### 🟢 포럼 전 (PRE)
- 통합 로그인 / 사전등록
- 익명 설정 & 개인정보 관리
- 참여 안내 / 행사 소개
- 앞으로 진행될 프로그램·일정표 확인
- 연사 및 아젠다 소개
- 지난 포럼 다시보기 (과거 포럼 확인)

### 🔴 포럼 중 (LIVE)
- 웨비나/오디오 (온라인 스트리밍, Zoom 연동)
- 온라인 대화
- 투표 (실시간)
- 의견(댓글) 등록
- 댓글 필터링 조회
- 실시간 성과 분석
- 라이브 블로그, 키 모먼트 표시
- **세션 상세 페이지에서 Zoom 시청, 토론, 화면 모드 전환(전체/반반/댓글), 댓글 게시 가능**
- **실명 세션 / 익명 세션 구분** (실명은 실명, 익명은 닉네임으로 댓글)
- 세션 요약 듣기, 연사 댓글 누르기, 자료보기/다운로드 가능

### 🔵 포럼 후 (POST)
- 설문조사
- 사후 성과 분석
- 지난 포럼 다시보기 및 하이라이트 세션 모아보기
- 올해의 포럼 내용을 '지난 포럼 보기'로 마이그레이션 후 관련 메뉴 닫음

### ⚙️ 공통 기능
- 참여 현황 & 토론현황 Bar & Pie 차트 인포그래픽
- 일자별·주제별 소개 페이지 및 필터링
- 카드 형태 UI
- 알림·공지 기능 (문자/카카오) 및 가이드 (매뉴얼 업로드/다운로드)
- 마이페이지 (나의 프로필 정보, 추천 세션, 알림 등록/세션 목록, 가입 취소)
- 인포그래픽 (핫 키워드 랭킹, 투표 참여 현황)
- 통합검색 (세션 중 / 연사 중 / 지난 포럼 세션 검색)
- 콘텐츠 매핑 로직 구현
- MyOOOO & 외부솔루션(Zoom) Interface 개발

## 3. 메뉴 구조 (PPT 슬라이드 기반)

### 포럼 전(PRE) 메뉴 구조

```
Home
├── 2026 OO포럼 소개
├── 현재 Live 세션
├── 오늘의 테마별 세션 목록
├── 인포그래픽
├── 올해의 연사 목록
├── 올해의 아젠다 목록
├── 나를 위한 추천 세션 목록
├── 2026 포럼 세션 목록 (비활성)
└── 2026 인기 영상 목록 (비활성)

상단 네비게이션:
[2026 프로그램] [2026 아젠다] [2026 연사] [지난 포럼 보기] [공지/문의] [마이페이지] [인포그래픽] [검색]
```

### 포럼 중(LIVE) 메뉴 구조
- 포럼 전과 동일하나 **현재 Live 세션, 오늘의 테마별 세션 목록** 등이 활성화
- 메인 포럼 진행 중 Live 세션 정보와 오늘의 일정, 참여 현황을 홈에서 바로 확인 가능

### 포럼 후(POST) 메뉴 구조
- **2026 프로그램/아젠다/연사** 메뉴 비활성화
- **지난 포럼 보기**와 **공지/문의**만 활성화
- 2026 포럼 세션 목록, 2025 인기 영상 목록 활성화

### 공통 하위 메뉴

| 메뉴 | 하위 페이지 |
|------|-----------|
| **2026 프로그램** | 일자별(테마별) 세션 목록, 일정 전체 보기 팝업 |
| **2026 아젠다** | 아젠다 카드 목록 |
| **2026 연사** | 주요 연사 목록, 아젠다별 연사 목록 |
| **지난 포럼 보기** | 연도별 세션 목록, 일정표(팝업) (2024~2026) |
| **공지 및 문의** | 공지사항 안내 게시판, 포럼 FAQ, 포럼 문의 번호/이메일 |
| **마이페이지** | 나의 프로필 정보, 추천 세션, 알림 등록 세션, 가입 취소 |
| **인포그래픽** | 핫 키워드 랭킹(순위), 투표 참여 현황(순위) |
| **통합 검색** | 세션 중 / 연사 중 / 지난 포럼 세션 검색 결과 |
| **푸터** | 개인 정보 취급 방침, 관련 사이트 목록 |

## 6. H/W 아키텍처

```
외부 참석자 ──→ ┌─────────────────────────────────────────────┐
관계사망        │           IBM Cloud Domain                  │
               │                                             │    Legacy Systems
임직원 ──→ CDN │──→ Load Balancer ──→ WAS(FO) Pod            │──→ SSO Server (TokTok)
               │                     (Container Engine)      │    LMS
               │                          │                  │
               │                     ┌────┴────┐             │    3rd Party Service
               │                     │ Storage │             │──→ MMS (SendGrid/Email)
               │                     └─────────┘             │
               │                                             │
               │         ZCP Cloud Domain                    │
               │    ┌──────────────────────────────┐         │
               │    │  MariaDB (Master-Slave)      │         │
               │    │  Redis (Cluster1, Cluster2)  │         │
               │    │  MMS Agent (Pod)             │         │
               │    └──────────────────────────────┘         │
               │                                             │
관리자 ──→     │  Load Balancer ──→ WAS(BO) Pod              │
               │                   (Container Engine)        │
               │                                             │
               │    Streaming Service (Panopto)              │
               │    Zoom 연동                                │
               └─────────────────────────────────────────────┘
```

| 구성요소 | 기술 스택 |
|----------|-----------|
| **아키텍처** | MSA (Microservice Architecture) 기반 |
| **클라우드** | IBM Cloud & AWS |
| **컨테이너** | Docker, OpenShift |
| **DB** | MariaDB (마스터-슬레이브 이중화) |
| **캐시** | Redis (클러스터 구성) |
| **WAS** | JBoss + Spring Boot (FO: 사용자용 / BO: 관리자용 분리) |
| **스트리밍** | Panopto, Zoom 연동 |
| **CDN** | CDN + Cloud Storage (AWS S3, IBM Cloud Object Storage) |
| **CI/CD** | Jenkins, ArgoCD |
| **모니터링** | Prometheus & Grafana, Kibana & Elasticsearch |
| **API Gateway** | Kong / Spring Cloud Gateway |
| **레거시 연동** | SSO(TokTok), LMS, MMS(SendGrid) |

## 7. 프로젝트 세부과업 (7단계)

### 1단계: 프로젝트 기획 및 설계
- 프로젝트 목표 및 요구사항 정의
- WBS 수립 및 일정 계획
- UI/UX 화면 설계 및 프로토타입 개발
- 대규모 트래픽을 고려한 MSA 아키텍처 설계
- 기술 스택 및 인프라 구성 정의 (IBM Cloud, AWS, OpenShift, Kubernetes 등)

### 2단계: 데이터 관리 및 전처리
- 이벤트 및 참가자 데이터 모델링
- 데이터 수집 및 전처리 파이프라인 구축
- RDBMS(PostgreSQL) 및 NoSQL(MongoDB, Redis) 설계
- 트랜잭션 관리 및 데이터 암호화 적용

### 3단계: 백엔드 개발 (MSA 기반)
- Spring Boot 기반 RESTful API 개발
- GraphQL & REST API 설계 및 구현
- 이벤트 등록 및 세션 관리 API 개발
- **WebSocket을 활용한 실시간 토론 및 투표 기능 구축**
- Spring Security 기반 JWT 인증 및 OAuth2 적용
- 서비스 간 통신 (gRPC, Kafka) 적용

### 4단계: 프론트엔드 개발
- **React & TailwindCSS** 기반 프론트엔드 대시보드 개발
- 이벤트 등록, 세션 참여, 실시간 토론 UI/UX 개발
- 실시간 차트 및 인포그래픽 기능 개발
- 다국어 지원 기능 개발
- SNS 연동 기능 구현 (SNS 알림 및 공유)

### 5단계: CI/CD 및 배포
- CI/CD 파이프라인 구축 (Jenkins, ArgoCD 활용)
- Docker 기반 컨테이너화
- OpenShift를 활용한 컨테이너 오케스트레이션
- API Gateway (Kong / Spring Cloud Gateway) 구성
- 배포 후 모니터링 시스템 구축 (Prometheus & Grafana)
- CDN 및 Cloud Storage 활용 성능 최적화

### 6단계: 테스트 및 최적화
- 대규모 트래픽 대응을 위한 부하 테스트 및 성능 테스트 진행
- 보안 점검 및 API Rate Limiting 적용
- 서버 로드 밸런싱 및 트래픽 분산 처리
- 데이터베이스 및 API 성능 최적화

### 7단계: 프로젝트 마무리 및 문서화
- 시스템 아키텍처 및 코드 문서화
- API 문서화 (Swagger / OpenAPI 3.0)
- 사용자 매뉴얼 및 기술 문서 작성
- 최종 프로젝트 보고서 및 발표 준비

## 9. 프로젝트 진행 시 주의사항

> [!IMPORTANT]
> 박성식 강사가 강조한 핵심 주의사항

- **범위 축소 필수**: 실제로 적지 않은 인력과 긴 기간이 소요된 프로젝트이므로, 교육 프로젝트 규모에 맞게 적정 범위 선정 필요
- **Zoom 연동(실시간 스트리밍) 제외 권고**: 구현 난이도가 높아 제외 추천
- **멘토링 활용**: 실제 구현 범위를 멘토링을 통해 결정 권장
- **프레임워크**: React 등 최신 프레임워크 사용 추천
- 다수의 사용자 접속시 **성능 저하 없어야** 하며, **MSA 아키텍처** 구성으로 진행
