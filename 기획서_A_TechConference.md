## 1. 🎯 프로젝트 개요

### 1-1. 프로젝트명
> **Tech Conference Hub** — 개발자를 위한 기술 컨퍼런스 관리 플랫폼

### 1-2. 프로젝트 배경 및 동기
> 왜 이 프로젝트를 하고 싶은가? 어떤 문제를 해결하고 싶은가?

- IBM이 실제 수행한 "Cloud Native 기반 통합 이벤트 관리 플랫폼"을 교육 프로젝트 규모에 맞게 재해석
- FECONF, DEVIEW, if(kakao) 같은 국내 기술 컨퍼런스가 매년 증가하지만, 통합 관리 플랫폼은 부족한 상황
- 개발자로서 직접 사용하고 싶은 서비스를 만들어 포트폴리오 가치를 극대화

### 1-3. 프로젝트 목표
> 이 프로젝트가 완성되면 어떤 가치를 제공하는가?

- **핵심 목표**: 기술 컨퍼런스의 세션 관리, 참가 신청, 실시간 소통을 지원하는 Cloud Native 웹 플랫폼 구축
- **세부 목표 1**: 컨퍼런스 정보 제공 및 세션별 등록/관리 기능 구현
- **세부 목표 2**: WebSocket 기반 실시간 Q&A 및 투표 기능 구현
- **세부 목표 3**: 참여 현황 데이터 시각화 대시보드 구현

### 1-4. 타겟 사용자
> 누가 이 서비스를 사용하는가?

| 사용자 유형 | 설명 | 주요 니즈 |
|------------|------|----------|
| 일반 참가자 | 컨퍼런스에 참석하는 개발자/IT 종사자 | 세션 탐색, 참가 신청, 실시간 Q&A 참여 |
| 연사(스피커) | 세션을 진행하는 발표자 | 세션 정보 등록, 프로필 관리, Q&A 응답 |
| 관리자 | 컨퍼런스를 기획·운영하는 운영진 | 세션/연사/참가자 관리, 통계 확인 |

## 3. 🔍 프로젝트 방향성

### 3-1. 핵심 컨셉 (한 줄 요약)
> **"개발자가 기술 컨퍼런스를 탐색하고, 세션에 참가하며, 실시간으로 소통하는 Cloud Native 플랫폼"**

### 3-2. 차별화 포인트

1. **실시간 Q&A & 투표** — WebSocket 기반으로 세션 중 실시간 질문·투표 가능
2. **데이터 시각화 대시보드** — 참여 현황, 인기 세션, 키워드 트렌드를 차트/인포그래픽으로 제공
3. **Cloud Native 아키텍처** — Docker 컨테이너화 + CI/CD 파이프라인으로 현업 수준의 배포 환경 구현

### 3-3. 참고 서비스 / 벤치마킹

| 서비스명 | URL | 참고할 점 | 개선할 점 |
|---------|-----|----------|----------|
| FECONF | https://feconf.kr | 깔끔한 세션 목록 UI, 연사 프로필 | 실시간 Q&A 기능 없음 |
| DEVIEW | https://deview.kr | 카테고리별 세션 분류, 다시보기 | 참가자 간 상호작용 부족 |
| 세계경제포럼(WEF) | https://www.weforum.org | 라이브 블로그, 키모먼트, 메뉴 구조 | 지나치게 대규모, 범위 축소 필요 |

## 6. 📐 화면 설계 (와이어프레임)

### 6-1. 주요 화면 목록

| # | 화면명 | 설명 | 비고 |
|---|-------|------|------|
| 1 | 메인 페이지 | 컨퍼런스 소개, 현재 라이브 세션, 추천 세션 | 히어로 섹션 + 카드 그리드 |
| 2 | 로그인/회원가입 | JWT 기반 인증 | 소셜 로그인은 제외 |
| 3 | 세션 목록 | 주제별/일자별 필터, 카드 UI | 검색 기능 포함 |
| 4 | 세션 상세 | 세션 정보 + 실시간 Q&A + 투표 | 핵심 페이지 |
| 5 | 연사 목록/상세 | 연사 프로필 카드 + 담당 세션 | |
| 6 | 통계 대시보드 | 참여 현황 차트, 인기 세션 랭킹 | 인포그래픽 |
| 7 | 마이페이지 | 내 프로필, 신청 세션 목록, 북마크 | |
| 8 | 관리자 대시보드 | 세션/연사/회원 관리, 통계 | 별도 레이아웃 |

### 6-2. 화면 흐름도 (User Flow)

```
[참가자 Flow]
메인 페이지 → 회원가입/로그인 → 세션 목록 → 세션 상세 → 참가 신청
                                                     → 실시간 Q&A 참여
                                                     → 실시간 투표 참여
                              → 연사 목록 → 연사 상세
                              → 통계 대시보드
                              → 마이페이지

[관리자 Flow]
관리자 로그인 → 관리자 대시보드 → 세션 관리 (CRUD)
                               → 연사 관리 (CRUD)
                               → 회원 관리
                               → 통계 조회
```

## 7. 📊 데이터 설계 (ERD 초안)

### 7-1. 주요 엔티티(테이블) 목록

| # | 엔티티명 | 설명 | 주요 속성 |
|---|---------|------|----------|
| 1 | User | 사용자 정보 | id, email, password, name, role(USER/SPEAKER/ADMIN), profile_image, created_at |
| 2 | Conference | 컨퍼런스 정보 | id, title, description, start_date, end_date, status(PRE/LIVE/POST), banner_image |
| 3 | Session | 세션 정보 | id, conference_id(FK), speaker_id(FK), title, description, category, start_time, end_time, max_capacity, current_count |
| 4 | Registration | 세션 참가 신청 | id, user_id(FK), session_id(FK), status(REGISTERED/CANCELLED), registered_at |
| 5 | Question | 실시간 Q&A | id, session_id(FK), user_id(FK), content, likes_count, is_answered, created_at |
| 6 | Vote | 실시간 투표 | id, session_id(FK), title, options(JSON), created_by(FK), is_active |
| 7 | VoteResponse | 투표 응답 | id, vote_id(FK), user_id(FK), selected_option, voted_at |
| 8 | Speaker | 연사 상세정보 | id, user_id(FK), bio, company, position, topics(JSON) |

### 7-2. 엔티티 관계

- User 1:N Registration — 한 사용자가 여러 세션에 참가 신청
- User 1:1 Speaker — 연사인 사용자는 연사 상세정보 보유
- Conference 1:N Session — 한 컨퍼런스에 여러 세션
- Session 1:N Question — 한 세션에 여러 Q&A 질문
- Session 1:N Vote — 한 세션에 여러 투표
- Vote 1:N VoteResponse — 한 투표에 여러 응답
