## 1. 개요

### 1-1. 목적
이 문서는 공공데이터포털(data.go.kr)의 **한국관광공사 국문 관광정보 서비스 API**를 활용하여 축제/행사 데이터를 주기적으로 수집·갱신하는 배치 프로세스의 설계를 정의합니다.

### 1-2. 용어 정의

| 용어 | 설명 |
|------|------|
| **배치(Batch)** | 사용자 요청 없이 시스템이 자동으로 주기적으로 실행하는 작업 |
| **동기화(Sync)** | 외부 API의 데이터를 우리 DB에 반영하는 작업 |
| **Upsert** | 데이터가 없으면 INSERT, 있으면 UPDATE하는 작업 |
| **contentId** | 한국관광공사 API에서 각 관광 콘텐츠를 식별하는 고유 ID |
| **contentTypeId** | 관광 타입 ID (축제/공연/행사 = `15`) |

### 1-3. 사용 API 정보

| 항목 | 내용 |
|------|------|
| **API명** | 한국관광공사_국문 관광정보 서비스_GW (KorService2) |
| **제공처** | 공공데이터포털 (data.go.kr) |
| **Base URL** | `https://apis.data.go.kr/B551011/KorService2` |
| **인증 방식** | 서비스 키 (serviceKey) — 쿼리 파라미터 방식 |
| **응답 형식** | JSON (`_type=json` 파라미터 추가) |
| **일일 트래픽 제한** | 일반 인증키 기준 1,000건/일 (활용 신청 시 확인 필요) |

## 3. BAT-001: 축제 데이터 전체 동기화

### 3-1. 기본 정보

| 항목 | 내용 |
|------|------|
| **배치 ID** | BAT-001 |
| **배치명** | 축제 데이터 전체 동기화 |
| **설명** | 한국관광공사 API에서 전국 축제/행사 데이터를 전체 조회하여 DB에 신규 등록 또는 갱신 |

| 조건 | 전략 | 사용 API |
|------|------|----------|
| 최초 실행 (batch_log 이력 없음) | 전체 동기화 | `searchFestival1` |

**showflag 처리:**
- `showflag=1`: 공개 데이터 → 정상 Upsert 처리
- `showflag=0`: 비공개/삭제 → DB에 있으면 `status='HIDDEN'`으로 변경

#### status 계산 로직

```java
// Java 비즈니스 로직
public FestivalStatus calculateStatus(LocalDate startDate, LocalDate endDate) {
    LocalDate today = LocalDate.now();
    
    if (today.isBefore(startDate)) {
        return FestivalStatus.UPCOMING;    // 예정
    } else if (!today.isAfter(endDate)) {
        return FestivalStatus.ONGOING;     // 진행 중
    } else {
        return FestivalStatus.ENDED;       // 종료
    }
}
```

### 3-5. Upsert 전략

```
┌──────────────────────────────────────────────────────────────┐
│  Upsert 판별 로직                                             │
│                                                              │
│  source_id(= API contentId) 기준으로:                         │
│                                                              │
│  1. DB에 해당 source_id가 없으면 → INSERT (신규 축제)          │
│                                                              │
│  2. DB에 해당 source_id가 있으면:                              │
│     → API의 modifiedtime과 DB의 api_modified_at 비교          │
│     → 다르면 UPDATE (변경된 필드만 덮어쓰기)                    │
│     → 같으면 SKIP (갱신 불필요)                               │
│                                                              │
│  3. showflag=0 (비공개/삭제)인 경우: (v1.1 추가)               │
```

```sql
        WHEN CURRENT_DATE >= start_date AND CURRENT_DATE <= end_date THEN 'ONGOING'
        WHEN CURRENT_DATE > end_date THEN 'ENDED'
```

| `INFO` | 수동 동기화 시작 | `[BAT-003] 수동 동기화 시작 (trigger: admin_id=1)` |
| `INFO` | 동기화 완료 | `[BAT-003] 완료: total=350, insert=12, update=8 (03:25)` |
| `WARN` | 중복 실행 차단 | `[BAT-003] 중복 실행 차단: 이미 동기화 진행 중` |
| `ERROR` | 동기화 실패 | `[BAT-003] 실패: 공공 API 호출 실패 (HTTP 502)` |

**관리자 페이지 모니터링:**
- 동기화 실행 중 실시간 진행률 표시 (현재 페이지 / 전체 페이지)
- 완료 시 결과 요약 표시 (inserted/updated/skipped/failed)
- 실패 시 에러 메시지 표시

## 6. 배치 실행 로그 테이블

### 6-1. batch_log 테이블 설계

```sql
CREATE TABLE batch_log (
    id              BIGSERIAL PRIMARY KEY,
    batch_id        VARCHAR(10)     NOT NULL,       -- 'BAT-001', 'BAT-002', 'BAT-003'
    batch_name      VARCHAR(100)    NOT NULL,       -- 배치명
    trigger_type    VARCHAR(10)     NOT NULL,       -- 'AUTO' / 'MANUAL'
    triggered_by    BIGINT          NULL,           -- MANUAL인 경우 관리자 user_id
    status          VARCHAR(10)     NOT NULL,       -- 'RUNNING' / 'SUCCESS' / 'FAILED'
    started_at      TIMESTAMP       NOT NULL,       -- 시작 시각
    finished_at     TIMESTAMP       NULL,           -- 종료 시각
    execution_time  VARCHAR(20)     NULL,           -- 소요 시간 (HH:mm:ss)
    total_count     INT             DEFAULT 0,      -- 총 처리 건수
    insert_count    INT             DEFAULT 0,      -- 신규 등록 건수
    update_count    INT             DEFAULT 0,      -- 갱신 건수
    skip_count      INT             DEFAULT 0,      -- 스킵 건수
    fail_count      INT             DEFAULT 0,      -- 실패 건수
    error_message   TEXT            NULL,           -- 에러 메시지 (실패 시)
    error_detail    TEXT            NULL,           -- 에러 상세 (스택 트레이스)
    created_at      TIMESTAMP       DEFAULT NOW()
);

-- 인덱스
CREATE INDEX idx_batch_log_batch_id ON batch_log(batch_id);
CREATE INDEX idx_batch_log_status ON batch_log(status);
CREATE INDEX idx_batch_log_started_at ON batch_log(started_at);
```

```
                 │ 업데이트 │       │ 동기화  │          │ 동기화  │

                      │                  │                     │
                      │                  │                     │
                      ▼                  ▼                     ▼
              status 변경 반영   API → DB 갱신 완료     API → DB 갱신 완료
```

## 8. Festival 테이블 보완 (ERD 변경사항)

기존 ERD에서 배치 설계 반영을 위해 **추가/변경이 필요한 컬럼**:

```sql
-- Festival 테이블 수정 (배치 설계 반영)
ALTER TABLE festival ADD COLUMN source_id      VARCHAR(20)  UNIQUE;   -- 공공 API contentId (Upsert 키)
ALTER TABLE festival ADD COLUMN source         VARCHAR(10)  DEFAULT 'API';  -- 'API' / 'MANUAL'
ALTER TABLE festival ADD COLUMN area_code      VARCHAR(5);            -- 지역 코드 (시/도)
ALTER TABLE festival ADD COLUMN sigungu_code   VARCHAR(5);            -- 시군구 코드
ALTER TABLE festival ADD COLUMN category_mid   VARCHAR(10);           -- 중분류 코드
ALTER TABLE festival ADD COLUMN category_sub   VARCHAR(10);           -- 소분류 코드
ALTER TABLE festival ADD COLUMN tel            VARCHAR(50);           -- 문의 전화
ALTER TABLE festival ADD COLUMN api_modified_at TIMESTAMP;            -- API 수정일 (변경 감지용)
ALTER TABLE festival ADD COLUMN updated_at     TIMESTAMP DEFAULT NOW(); -- DB 최종 갱신일

-- v1.1 추가: 상세 정보 보강 필드
ALTER TABLE festival ADD COLUMN overview       TEXT;                    -- 축제 개요 설명
ALTER TABLE festival ADD COLUMN event_place    VARCHAR(200);           -- 행사 장소명
ALTER TABLE festival ADD COLUMN play_time      VARCHAR(200);           -- 공연/행사 시간
ALTER TABLE festival ADD COLUMN program        TEXT;                    -- 행사 프로그램
ALTER TABLE festival ADD COLUMN use_fee        VARCHAR(200);           -- 이용 요금
ALTER TABLE festival ADD COLUMN sponsor        VARCHAR(100);           -- 주최 기관
```

| 컬럼명 | 추가 사유 |
|--------|----------|
| `source_id` | 공공 API contentId를 저장하여 Upsert 기준 키로 사용 |
| `source` | API 데이터 / 수기 등록 데이터 구분 (멘토 피드백 반영) |
| `area_code` | 지역별 필터링 및 통계에 활용 |
| `sigungu_code` | 시군구 단위 필터링 |
| `category_mid/sub` | 상세 카테고리 필터링 |
| `tel` | 축제 문의 전화 |
| `api_modified_at` | API 수정일과 비교하여 변경 여부 판별 (불필요한 UPDATE 방지) |
| `updated_at` | DB 갱신 시각 추적 |
| `overview` | 축제 상세 설명 — detailCommon1 API **(v1.1)** |
| `event_place` | 행사 장소명 — detailIntro1 API **(v1.1)** |
| `play_time` | 공연/행사 시간 — detailIntro1 API **(v1.1)** |
| `program` | 행사 프로그램 내용 — detailIntro1 API **(v1.1)** |
| `use_fee` | 이용 요금 정보 — detailIntro1 API **(v1.1)** |
