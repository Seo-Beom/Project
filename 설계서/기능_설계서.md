## 1. 개요

### 1-1. 목적
이 문서는 프로젝트의 모든 API 엔드포인트를 **컨트롤러 단위**로 묶고, 각 컨트롤러가 호출하는 **서비스(UseCase)** 목록을 정리합니다.

### 1-2. 설계 원칙
- **헥사고날 아키텍처**: Controller → UseCase(Port IN) → Service → Port OUT → Adapter
- **공통 응답 포맷**: 모든 API는 `ApiResponse<T>` 형태로 응답
- **인증**: Spring Security + JWT, 비회원/회원/관리자 3단계 권한

### 1-3. API ID 규칙

> **팀 공통 규칙**: `API_{도메인코드}_{4자리 순번}`

| 도메인 코드 | 영역 | 설명 |
|------------|------|------|
| `AUTH` | 인증 | 회원가입, 로그인 |
| `USR` | 사용자 | 프로필 관리 |
| `FES` | 축제 | 축제 조회, 검색, 지도, 달력, 즐겨찾기 |
| `REV` | 리뷰 | 리뷰/별점 CRUD |
| `COM` | 커뮤니티 | 게시글/댓글 CRUD |
| `NTC` | 공지사항 | 공지 조회 |
| `RPT` | 신고 | 신고 접수 |
| `ADM` | 관리자 | 관리자 전용 기능 |

### 1-4. 공통 응답 포맷

```json
// 성공
{ "success": true, "data": { ... }, "error": null }

// 실패
{ "success": false, "data": null, "error": { "code": "AUTH_001", "message": "인증이 필요합니다" } }

// 페이징
{ "success": true, "data": {
    "content": [ ... ],
    "page": 1, "size": 10, "totalElements": 150, "totalPages": 15
  }, "error": null }
```

### 1-5. 공통 헤더

| 헤더 | 값 | 필수 | 설명 |
|------|-----|------|------|
| `Content-Type` | `application/json` | ✅ | 요청/응답 형식 |
| `Authorization` | `Bearer {JWT}` | 🔒 회원/관리자 | 인증 토큰 |

### 1-6. 인증 레벨 범례

| 표기 | 의미 |
|------|------|
| ✅ 비회원 | 인증 없이 접근 가능 |
| 🔒 회원 | JWT 필수 (role=USER) |
| 🔐 관리자 | JWT 필수 (role=ADMIN) |

### API_AUTH_0001: 회원가입

| 항목 | 내용 |
|------|------|
| **API ID** | API_AUTH_0001 |
| **엔드포인트** | `POST /api/auth/register` |
| **UseCase** | `RegisterUseCase` |
| **Service** | `AuthService.register()` |
| **인증** | ✅ 불필요 |

**Request Body:**
```json
{
  "email": "user@example.com",
  "password": "Password123!",
  "passwordConfirm": "Password123!",
  "nickname": "축제러버"
}
```

**유효성 검사:**

| 필드 | 규칙 |
|------|------|
| email | 이메일 형식, 중복 불가, 최대 100자 |
| password | 8~20자, 영문+숫자+특수문자 조합 |
| passwordConfirm | password와 일치 |
| nickname | 2~20자, 중복 불가 |

**Response (201):**
```json
{ "success": true, "data": { "userId": 1, "email": "user@example.com", "nickname": "축제러버" } }
```

**에러:** 400 유효성 실패 / 409 이메일·닉네임 중복

## 11. Spring Security URL 패턴 설정

```java
// SecurityConfig.java — URL별 인가 규칙 요약
http.authorizeHttpRequests(auth -> auth
    // 인증 불필요 (비회원 허용)
    .requestMatchers(HttpMethod.POST, "/api/auth/**").permitAll()
    .requestMatchers(HttpMethod.GET, "/api/festivals/**").permitAll()
    .requestMatchers(HttpMethod.GET, "/api/reviews").permitAll()
    .requestMatchers(HttpMethod.GET, "/api/community/**").permitAll()
    .requestMatchers(HttpMethod.GET, "/api/notices/**").permitAll()
    
    // 회원 전용
    .requestMatchers("/api/profile/**").hasRole("USER")
    .requestMatchers(HttpMethod.POST, "/api/reviews").hasRole("USER")
    .requestMatchers(HttpMethod.PUT, "/api/reviews/**").hasRole("USER")
    .requestMatchers(HttpMethod.DELETE, "/api/reviews/**").hasRole("USER")
    .requestMatchers("/api/favorites/**").hasRole("USER")
    .requestMatchers(HttpMethod.POST, "/api/community/**").hasRole("USER")
    .requestMatchers(HttpMethod.PUT, "/api/community/**").hasRole("USER")
    .requestMatchers(HttpMethod.DELETE, "/api/community/**").hasRole("USER")
    .requestMatchers(HttpMethod.POST, "/api/reports").hasRole("USER")
    
    // 관리자 전용
    .requestMatchers("/api/admin/**").hasRole("ADMIN")
    
    // 그 외 인증 필요
    .anyRequest().authenticated()
);
```
