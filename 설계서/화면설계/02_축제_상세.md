## 레이아웃 구성 (2컬럼)

### 좌측 영역 (60%)

| 영역 | 구성 요소 |
|------|---------|
| 히어로 이미지 | API `firstimage` 대형 이미지 |
| 축제명 | `title` 대형 볼드 텍스트 |
| 메타 정보 | 📅 기간 \| 📍 위치 \| 🔖 북마크 아이콘 |
| ① 상세 설명 | `overview` 텍스트 + 서브 이미지 썸네일 |

### 우측 영역 (40%)

| 영역 | 구성 요소 |
|------|---------|
| ④ 정보 박스 | 위치, 기간, 문의처 (`tel`) 테이블 |
| ② 별점/만족도 | 평균 점수 (4.5) + 별점 분포 바 차트 (5점~1점) |

### 하단 (전체 너비)

| 영역 | 구성 요소 |
|------|---------|
| 후기 섹션 | 사용자 아바타 + 닉네임 + 별점 + 리뷰 내용 + 이미지 |
| ③ 페이지네이션 | < 1 2 3 > (리뷰 목록 페이징) |

## 인터랙션

| 액션 | 동작 | 조건 |
|------|------|------|
| 북마크 아이콘 클릭 | 즐겨찾기 추가/해제 토글 | 🔒 로그인 필수 |

| 페이지 상세 진입 | `view_count` +1 자동 증가 | 세션/쿠키 중복 방지 |
| 뒤로가기 | 이전 목록 페이지로 복귀 | — |

## API 호출

```
# 축제 상세
GET /api/festivals/{festivalId}

# 리뷰 목록
GET /api/reviews?festivalId={festivalId}&page=1&size=5&sort=latest

# 즐겨찾기 추가
POST /api/favorites  { "festivalId": 1 }

# 즐겨찾기 해제
DELETE /api/favorites/{favoriteId}
```

---

## 비즈니스 규칙

- ENDED 상태 축제만 리뷰 작성 가능
- 축제당 1인 1리뷰 제한
- 조회수는 동일 세션에서 중복 증가 방지
